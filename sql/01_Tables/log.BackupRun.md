Stores execution telemetry for every backup operation performed by the framework.

This table provides operational traceability, execution auditing, and performance metrics for backup processes executed through the framework's automated backup procedures.

Each record represents a single backup execution event.

### **a) Structure:**
| Column | Type | Description |
|----------|------|-------------|
| BackupRunID | bigint | Unique identifier of the backup execution record. |
| StartedAt | datetime2 | Timestamp indicating when the backup operation started. |
| EndedAt | datetime2 | Timestamp indicating when the backup operation completed. |
| DatabaseName | sysname | Name of the database that was backed up. |
| BackupType | varchar | Type of backup performed (`FULL`, `DIFF`, `LOG`). |
| TierID | tinyint | Logical tier classification used by the backup strategy to determine backup frequency and retention policy. |
| PathType | varchar | Storage path classification used to determine the destination directory for the backup file (e.g., `PRIMARY`, `SECONDARY`). |
| PrimaryFile | nvarchar | Full path of the primary backup file generated by the operation. |
| SecondaryFile | nvarchar | Full path of the mirrored backup file when backup mirroring is enabled. |
| UsedMirror | bit | Indicates whether backup mirroring was used during the operation. |
| WithChecksum | bit | Indicates whether the backup was executed with the `CHECKSUM` option enabled. |
| WithCompression | bit | Indicates whether backup compression was enabled. |
| IsCopyOnly | bit | Indicates whether the backup was executed using the `COPY_ONLY` option. |
| VerifyRequested | bit | Indicates whether a `RESTORE VERIFYONLY` validation was requested after backup completion. |
| VerifySucceeded | bit | Indicates whether the backup verification operation succeeded. |
| BackupSizeBytes | bigint | Size of the backup before compression, measured in bytes. |
| CompressedSizeBytes | bigint | Size of the compressed backup file written to disk. |
| Succeeded | bit | Indicates whether the backup operation completed successfully. |
| ErrorNumber | int | SQL Server error number recorded if the backup operation failed. |
| ErrorMessage | nvarchar | Detailed error message captured during backup failure. |
| HostName | sysname | Host machine where the backup operation was executed. |
| InstanceName | sysname | SQL Server instance name where the backup was performed. |
| SqlVersion | nvarchar | SQL Server version of the instance performing the backup. |
| CorrelationID | uniqueidentifier | Identifier used to correlate this backup execution with other operations within the framework. |
